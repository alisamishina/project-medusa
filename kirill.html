David with the Head of Goliath

I chose the painting called David with the Head of Goliath. First, I try to find it in the ARCO ontology using only the name: 


PREFIX rdfs: http://www.w3.org/2000/01/rdf-schema#


PREFIX arco: https://w3id.org/arco/ontology/arco/

SELECT DISTINCT ?painting ?label
WHERE {
?painting a arco:HistoricOrArtisticProperty ;
rdfs:label ?label .
FILTER(REGEX(?label, "david", "i"))
FILTER(REGEX(?label, "goliath", "i"))
}

LIMIT 100

If we write only the name of the painting, there are still many results, because several artworks contain the names "David" or "Goliath". So I narrow it down by adding the author, Caravaggio, and the language of the label - English: 


PREFIX rdfs: http://www.w3.org/2000/01/rdf-schema#

PREFIX a-cd: https://w3id.org/arco/ontology/context-description/

SELECT DISTINCT ?painting ?label ?authorName
WHERE {
?painting rdfs:label ?label ;
a-cd:hasAuthor ?author .
?author rdfs:label ?authorName .

FILTER(regex(?label, "david", "i"))
FILTER(regex(?label, "goliath", "i"))
FILTER(regex(?authorName, "caravaggio", "i"))
FILTER(lang(?label) = "en")
}

LIMIT 10

The result of the query is very good: it returns only a few links, all referring to the correct artwork. Some entries list the author simply as "Caravaggio", while others use the full name "Merisi Michelangelo detto Caravaggio". This confirms that we have identified the correct resource in the Knowledge Graph.


Next, I check whether the painting already has subject information in ArCo. To do this, I query the ontology for the property a-cd:hasSubject: 


PREFIX rdfs: http://www.w3.org/2000/01/rdf-schema#

PREFIX a-cd: https://w3id.org/arco/ontology/context-description/

SELECT DISTINCT ?subject ?subjectLabel
WHERE {
?painting rdfs:label ?label .
FILTER(regex(?label, "david", "i"))
FILTER(regex(?label, "goliath", "i"))

OPTIONAL {
?painting a-cd:hasSubject ?subject .
?subject rdfs:label ?subjectLabel .
}
}

LIMIT 20

The result shows that no subjects are associated with the painting. This confirms the GAP: the artwork lacks semantic subject information such as "David", "Goliath", or "Decapitation".


To enrich the Knowledge Graph, I use an LLM to extract the main subjects of the painting. The model identifies:


  • David
  • Goliath
  • The severed head
  • The act of decapitation
  • A symbolic interpretation: Caravaggio's self-portrait as Goliath

Finally, I ask the LLM to convert this information into RDF triples using the ArCo ontology: 


@prefix ex: <http://example.org/> .

@prefix arco:  .
@prefix a-cd:  .
@prefix rdfs:  .

ex:David_Goliath a arco:CulturalProperty ;
a-cd:hasSubject ex:David ;
a-cd:hasSubject ex:Goliath ;
a-cd:hasSubject ex:Decapitation ;
a-cd:hasInterpretation ex:Autoportrait ;
a-cd:iconclassCode "71D123" .

ex:David a a-cd:Subject ; rdfs:label "David" .
ex:Goliath a a-cd:Subject ; rdfs:label "Goliath" .
ex:Decapitation a a-cd:Subject ; rdfs:label "Decapitation of Goliath" .
ex:Autoportrait a a-cd:Interpretation ; rdfs:label "Caravaggio's self-portrait as Goliath" .

These triples enrich the painting with missing semantic information, making it more complete and more useful for cultural heritage analysis.
